1. 시스템 아키텍처

5. 수질 등급 기준

출처: 환경정책기본법 시행규칙 [별표 1] — 하천 생활환경기준. 이 표의 수치는 apps/web/src/lib/grade.ts 의 THRESHOLDS 및 docs/grading.md 와 1:1 일치한다 (개정 시 3 파일 동시 변경, ADR-0003).

등급 라벨 pH DO (mg/L) TOC (mg/L) T-P (mg/L)
Ia매우 좋음6.5–8.5≥ 7.5≤ 2≤ 0.02
Ib좋음6.5–8.5≥ 5.0≤ 3≤ 0.04
II약간 좋음6.5–8.5≥ 5.0≤ 4≤ 0.1
III보통6.5–8.5≥ 5.0≤ 5≤ 0.2
IV약간 나쁨6.0–8.5≥ 2.0≤ 6≤ 0.3
V나쁨6.0–8.5≥ 2.0≤ 8≤ 0.5
VI매우 나쁨—< 2.0> 8> 0.5
측정불가—모든 graded 항목이 점검중 → 등급 산정 불가

종합 등급 산출 (worst-of)

flowchart TD
    M[측정값 row]
    M --> ph["pH grade"]
    M --> do["DO grade"]
    M --> toc["TOC grade"]
    M --> tp["T-P grade"]
    ph --> W{worst-of}
    do --> W
    toc --> W
    tp --> W
    W --> C[종합 등급]

    M --> ph2["페놀 > 0 ?"]
    M --> cn2["시안 > 0 ?"]
    ph2 --> T[독성 별도 표기]
    cn2 --> T
  • 4개 graded 항목 (pH, DO, TOC, T-P) 중 가장 나쁜 등급 채택.
  • null 항목은 skip. 4개 모두 null 이면 'unmeasurable' → "측정불가".
  • T-N은 하천 기준 부재(호소 기준만 존재) → 측정값은 표시하되 등급 부여 안 함.
  • 페놀·시안은 사람의 건강보호기준 → 종합 등급에 포함 X. 검출 / 불검출 텍스트 우선 (Phase 1 단순 규칙 v > 0 → 검출; ADR-0002).

6. Seoul Open API 사전

엔드포인트 패턴: http://openapi.seoul.go.kr:8088/{KEY}/json/WPOSInformationTime/{START}/{END}/

  • 1회 최대 1,000건, 호출 횟수 제한 없음
  • 행 정렬: 관측 결과 최신 → 과거 (2026-05-19 확인, 문서화되지 않음 → 클라이언트 정렬 필수)
  • CORS 헤더 없음 → 브라우저 직접 호출 불가
  • 라이선스: 공공누리 1유형 (출처표시), 운영기관: 서울시 보건환경연구원
API 필드한글단위정규화 키비고
YMD날짜YYYYMMDDtimestampHR과 함께 KST Date
HR시각HH:MMtimestamp—
MSRSTN_NM측정소명—station탄천 / 중랑천 / 안양천 / 선유
WATT수온°Cwatt등급 산정 안 함
TOT_PHpH—ph등급 산정 (range)
TOT_DO용존산소mg/LdoO2등급 산정 (floor)
TOT_N총질소mg/LtotalN하천 기준 부재 → 등급 X
TOT_TP총인 (T-P)mg/LtotalP등급 산정 (ceiling)
TOT_OC총유기탄소 (TOC)mg/LtotalOC등급 산정 (ceiling)
PHNL페놀mg/Lphenol독성 별도 표기
CN시안mg/Lcyanide독성 별도 표기

데이터 이슈

  • "점검중" 문자열이 숫자 필드에 섞임 → 정규화에서 null. 2026-05-19 smoke test: 500행 중 192행이 최소 1개 셀에 포함.
  • 실패 코드 (자주): INFO-100 인증키 오류, ERROR-336 1,000건 초과. 전체 코드는 docs/seoul-api.md 참고.
